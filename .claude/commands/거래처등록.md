# ECOUNT 거래처 등록 (단건/복수건)

사용자가 제공하는 거래처 정보를 ECOUNT API로 등록합니다. 최대 300건까지 한 번에 등록 가능합니다.

## 입력 형식
사용자가 아래와 같이 여러 건의 데이터를 제공합니다:
```
거래처코드, 회사명, 대표자, 업태, 종목, 전화번호, 이메일
119-81-12345, 휴먼세크, 홍길동, 도소매, 보안시스템, 010-1234-5678, test@test.com
00003, 테스트업체, 김철수, 서비스업, 소프트웨어, 02-9876-5432,
```

각 항목 매핑:
- **거래처코드** → BUSINESS_NO (필수, ERP 거래처코드로 사용됨)
- **회사명** → CUST_NAME (필수)
- **대표자** → BOSS_NAME
- **업태** → UPTAE
- **종목** → JONGMOK
- **전화번호** → TEL
- **이메일** → EMAIL
- **주소** → ADDR
- **우편번호** → POST_NO
- **팩스** → FAX
- **모바일** → HP_NO
- **구분** → GUBUN (11: 일반거래처, 13: 관세사거래처. 기본값: 11)
- **거래처코드구분** → G_GUBUN (01: 사업자등록번호, 02: 주민등록번호, 03: 외국인. 기본값: 01)
- **담당자코드** → EMP_CD
- **적요** → REMARKS
- **검색창내용** → REMARKS_WIN
- **거래처그룹1** → CUST_GROUP1
- **거래처그룹2** → CUST_GROUP2

## 실행 방법

### 1단계: .env에서 인증 정보 읽기
```
cat D:/work/.env
```

### 2단계: 로그인
```
curl -s -X POST "https://oapiaa.ecount.com/OAPI/V2/OAPILogin" \
  -H "Content-Type: application/json" \
  -d '{"COM_CODE":"<COM_CODE>","USER_ID":"<USER_ID>","API_CERT_KEY":"<API_CERT_KEY>","LAN_TYPE":"ko-KR","ZONE":"AA"}'
```
응답에서 SESSION_ID를 추출합니다.

### 3단계: 거래처 등록 (복수건 일괄)
CustList 배열에 여러 건을 넣어 한 번에 전송합니다.
```
curl -s -X POST "https://oapiaa.ecount.com/OAPI/V2/AccountBasic/SaveBasicCust?SESSION_ID=<SESSION_ID>" \
  -H "Content-Type: application/json; charset=utf-8" \
  --data-binary @- <<'EOF'
{"CustList":[
  {"BulkDatas":{"BUSINESS_NO":"<거래처코드1>","CUST_NAME":"<회사명1>","BOSS_NAME":"<대표자1>","UPTAE":"<업태1>","JONGMOK":"<종목1>","TEL":"<전화번호1>","EMAIL":"<이메일1>"}},
  {"BulkDatas":{"BUSINESS_NO":"<거래처코드2>","CUST_NAME":"<회사명2>","BOSS_NAME":"<대표자2>","UPTAE":"<업태2>","JONGMOK":"<종목2>","TEL":"<전화번호2>","EMAIL":"<이메일2>"}}
]}
EOF
```

## 주의사항
- **한글 데이터**: 반드시 `charset=utf-8` + `--data-binary` 사용 (안 하면 한글 깨짐)
- **거래처코드**: BUSINESS_NO 필드에 사업자등록번호 또는 자체 코드 입력
- **복수건**: CustList 배열에 최대 300건까지 가능
- **중복 불가**: 이미 등록된 거래처코드는 등록 불가 (중복 에러 발생)
- **실서버**: `https://oapiaa.ecount.com` (테스트서버: `https://sboapiaa.ecount.com`)
- 빈 값인 선택 항목은 생략 가능 (빈 문자열로 보내도 무방)

## 응답 확인
- `SuccessCnt`: 성공 건수
- `FailCnt`: 실패 건수
- `SlipNos`: 전표번호 배열 (거래처코드-회사명 형태)
- 실패 시 `ResultDetails`의 `TotalError`에서 원인 확인
