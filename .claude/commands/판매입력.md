# ECOUNT 판매 입력 (단건/복수건)

사용자가 제공하는 판매 정보를 ECOUNT API로 등록합니다. 여러 건을 한 번에 입력할 수 있습니다.

## 입력 형식
사용자가 아래와 같이 여러 건의 데이터를 제공합니다:
```
거래처코드, 품목코드, 규격, 수량, 단가
119-81-71206, 90119, 삼보테크노현장 사무실 기계공사, 1, 50000
119-81-71206, 90120, 부품 교체, 2, 30000
```

각 항목 매핑:
- **거래처코드** → CUST (주의: CUST_CD 아님)
- **품목코드** → PROD_CD (필수)
- **규격** → SIZE_DES
- **수량** → QTY (필수)
- **단가** → PRICE + UNIT_PRICE (둘 다 동일값, 필수)
- **공급가액** → SUPPLY_AMT (제공 안 되면 수량×단가로 자동 계산)
- **부가세** → VAT_AMT (제공 안 되면 공급가액×10%로 자동 계산)
- **창고코드** → WH_CD (기본값 "100")
- **전표일자** → IO_DATE (기본값 오늘 YYYYMMDD)

## 실행 방법

### 1단계: .env에서 인증 정보 읽기
```
cat D:/work/.env
```

### 2단계: 로그인
```
curl -s -X POST "http://sboapiaa.ecount.com/OAPI/V2/OAPILogin" \
  -H "Content-Type: application/json" \
  -d '{"COM_CODE":"<COM_CODE>","USER_ID":"<USER_ID>","API_CERT_KEY":"<API_CERT_KEY>","LAN_TYPE":"ko-KR","ZONE":"AA"}'
```
응답에서 SESSION_ID를 추출합니다.

### 3단계: 판매 등록 (복수건 일괄)
SaleList 배열에 여러 건을 넣어 한 번에 전송합니다. UPLOAD_SER_NO가 같으면 하나의 전표로 묶입니다.
```
curl -s -X POST "http://sboapiaa.ecount.com/OAPI/V2/Sale/SaveSale?SESSION_ID=<SESSION_ID>" \
  -H "Content-Type: application/json; charset=utf-8" \
  --data-binary @- <<'EOF'
{"SaleList":[
  {"Line":"0","BulkDatas":{"IO_DATE":"<일자>","WH_CD":"100","PROD_CD":"<품목1>","SIZE_DES":"<규격1>","QTY":"<수량1>","UNIT_PRICE":"<단가1>","PRICE":"<단가1>","SUPPLY_AMT":"<공급가액1>","VAT_AMT":"<부가세1>","CUST":"<거래처1>","UPLOAD_SER_NO":"1"}},
  {"Line":"1","BulkDatas":{"IO_DATE":"<일자>","WH_CD":"100","PROD_CD":"<품목2>","SIZE_DES":"<규격2>","QTY":"<수량2>","UNIT_PRICE":"<단가2>","PRICE":"<단가2>","SUPPLY_AMT":"<공급가액2>","VAT_AMT":"<부가세2>","CUST":"<거래처2>","UPLOAD_SER_NO":"1"}}
]}
EOF
```

## 주의사항
- **한글 데이터**: 반드시 `charset=utf-8` + `--data-binary` 사용 (안 하면 한글 깨짐)
- **단가**: `UNIT_PRICE`와 `PRICE` 둘 다 같은 값으로 넣어야 반영됨
- **거래처**: 필드명은 `CUST` (CUST_CD 아님)
- **복수건**: SaleList 배열에 최대 300건까지 가능. Line은 0부터 순번
- **전표 묶음**: 같은 UPLOAD_SER_NO 값이면 하나의 전표로 묶임
- **테스트 서버**: `http://sboapiaa.ecount.com` (운영 전환 시 URL과 .env의 API키만 변경)
- 공급가액/부가세가 제공되지 않으면 자동 계산: 공급가액=수량×단가, 부가세=공급가액×10%

## 응답 확인
- `SuccessCnt`: 성공 건수
- `FailCnt`: 실패 건수
- `SlipNos`: 전표번호 배열
- 실패 시 `ResultDetails`의 `TotalError`에서 원인 확인
